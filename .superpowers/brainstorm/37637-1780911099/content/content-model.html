What Lives Where

A clear picture of what you manage in Sanity, what Supabase stores automatically, and what Resend handles.

🟠 Sanity Studio
You manage this
Trips
  • Title, description
  • Destination, dates
  • Price (deposit + total)
  • Max group size
  • Itinerary (day by day)
  • Photos
  • Difficulty level
  • Status (open / full / archived)
Applications
  • Applicant name + email
  • Which trip
  • Form answers
  • Status (pending / approved / rejected)
Bookings
  • Linked to application + trip
  • Deposit paid? ✓
  • Balance paid? ✓
  • Balance due date
Pages
  • Homepage text + hero
  • About page
  • FAQ items
🟢 Supabase
Automatic — no action needed
User Accounts
  • Email address
  • Password (encrypted)
  • Account created date
  • Login sessions
Traveler Profiles
  • First + last name
  • Profile photo
  • Age / nationality
  • Short bio
  • Travel experience
  • Languages spoken
Portal Access
  • Which trip they're on
  • Access granted date
  • Linked to Sanity booking
📧 Resend
Fully automatic emails
To you (info@chasingted.com)
  • New application received
To the customer
  • Application confirmed
  • Application approved
    + bank transfer details
  • Application rejected
    (polite decline)
  • Deposit received
    + portal login invite
  • Balance reminder
    (X weeks before trip)
📊 Google Tag Manager — installed directly in Next.js. Tracks: application submitted, portal login, WhatsApp click, contact form submit. Your existing GTM container (GTM-P7V9CJF3) already has these tags built — we just add the code snippet to the new site.
